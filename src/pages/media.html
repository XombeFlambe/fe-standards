Media

All media files should be saved in the media directory. The base sub-directories include fonts, images, videos, and uploads.

  • Fonts
  • Images
  • Videos
  • Uploads

Fonts

All font files should be stored in the media/fonts directory (This includes icon font files). It is best practice to always have a browser safe fallback font which will protect the project from file corruption and font server failures. Browsers use default fonts and browser safe fonts to style the text content of a website. Although the list of browser safe fonts is growing, designers tend to find the selection limiting. There are two popular practices used to expand the list of fonts that a project can use:

Font Face

Font face allows the developer to pull font files from a hosted source and/or the project directly. There is no standard font file for browsers yet, so multiple files types of the same font are necessary to import (.eot, .woff, .ttf, etc.). In the example below, the font files are stored in the media/fonts directory.

Example

src/media/fonts

webfont.eot
webfont.est?iefix
webfont.woff2
webfont.woff
webfont.ttf
webfont.svg#svgFontName

SCSS

/* Place in the css/scss/base/_fonts.scss file */
@font-face {
    font-family: 'MyWebFont';
    src: url('../media/fonts/webfont.eot'); /* IE9 Compat Modes */
    src: url('../media/fonts/webfont.eot?#iefix') format('embedded-opentype'), /* IE6-IE8 */
         url('../media/fonts/webfont.woff2') format('woff2'), /* Super Modern Browsers */
         url('../media/fonts/webfont.woff') format('woff'), /* Pretty Modern Browsers */
         url('../media/fonts/webfont.ttf')  format('truetype'), /* Safari, Android, iOS */
         url('../media/fonts/webfont.svg#svgFontName') format('svg'); /* Legacy iOS */
}

/* Body is an example placement, font-family property can be placed on any selector */
body {
    font-family: 'MyWebFont', Fallback, sans-serif;
}

Google Fonts

Google Fonts allow the developer quick access to a large library of fonts without having to store font files in the project. The benefits of this approach are faster font integration, not storing font files directly in the project, and Google Fonts are free to use. The con is that the project fonts will be affected if Google Fonts server goes down, which is highly unlikely. Google Fonts also has a page load gauge helping developers make better decisions on font download selections. This project and many current Wiley projects are using the Google Font approach.

Example

HTML

<!-- PLACE IN HEAD, GROUP FONT FILE PATHS TOGETHER -->
<link href='https://fonts.googleapis.com/css?family=Lato:400,700,300|Montserrat:400,700' rel='stylesheet' type='text/css'>

CSS

/* Body is an example placement, font-family property can be placed on any selector */
body {
    font-family: 'Lato', Fallback, sans-serif;
}

Images

There are many image file types that are browser compatible, but three (jpg, gif, png) are browser safe and one is nearly browser safe (svg) on the top browsers (Chrome, IE, Firefox, Safari, and Opera). Each of these file types have pros and cons as listed below.

  • JPG
    • Pros: Displays photos and optimizes photo file size better than other file types.
    • Cons: Displaying a jpg larger or smaller than its native size can result in pixelization. Does not allow transparency. Still popular for its great photo compression, but due to its lack of flexibility and transparency, is being replaced by the png.
  • GIF
    • Pros: Displays graphics and optimizes graphic file size better than jpg or png files. Can be used for simple animations.
    • Cons: Does not allow transparency. Animation can not be controlled. Losing popularity and becoming obsolete by the png and svg file types.
  • PNG
    • Pros: Works great with both photo and graphic images, surpassing both jpg and gif usage. Allows transparency (great for images with transparent backgrounds and opacity levels). Displays responsive images better than jpg and gif files, including the ability to enlarge png file sizes larger than its native size (great for using site background images).
    • Cons: Larger file sizes (roughly 3x) than jpg for photo images and gif for graphic images.
  • SVG
    • Pros: Highly customizable, it is quickly becoming the file of choice for logos, graphics, animated graphics and icons. Can be placed inside an image tag or directly inline with the <svg> tag (html compatible code). Optimal sizing for vector images, even better than gifs. Completely scalable, one size fits all. Allows transparency.Can be animated either with css or svg animations. Can be paired with Canvas, a powerful javascript animation library.
    • Cons: Only used for vector layouts. The svg file is still relatively new technology, so there are some browser compatibility issues (for example IE10 and below have an svg height bug).

Examples

  • JPG, 75kb
  • PNG, 175kb
  • GIF, 10.11kb
  • PNG, 34.5kb
  • PNG, 10.1kb
  • SVG, 5kb

Videos

Videos can be stored in the project's media/videos directory, but it is best practice to use a video sharing site (Youtube, Vimeo, etc.) or video marketing platform. Wiley used Youtube in the past but currently stores it's videos with the video marketing platform, Vidyard. A user account is needed to access the Wiley Vidyard account (contact your manager if you do not have access). There are three ways to share a Vidyard video: sharing page, inline embed, and lightbox embed. Light box embedding is the Wiley standard, which creates a video modal experience. Modal styling should remain consistent throughout the project.

Pros of using Vidyard:

  • Videos not stored directly in project, greatly reducing website load times
  • Built in embedding
  • Analytics

Uploads

Document files meant to be uploaded by the user should be stored in the media/uploads directory. Common file types include .pdf, .xlt, .doc, etc. PDF files are browser compatible, and the link should use the attribute target="_blank" to open a new window that displays the PDF. Non browser compatible file links should not use the target attribute, as they default to opening a download window.

Examples
  • Link to a browser compatible file.
  • Link to a browser non-compatible file.

HTML

<a href="./media/uploads/363FAQ.pdf" target="_blank">Link to a browser compatible file.</a>
<a href="./media/uploads/LeaderProcessLaunch.doc">Link to a browser non-compatible file.</a>
