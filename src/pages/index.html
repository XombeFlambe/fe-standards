Welcome to the front end web development standards for Wiley websites and applications — a collection of best practices, common object/module examples, tutorials and resources.

FE web development is in a constant state of flux, with an accelerating pace of change due to technological advancements, improved focus on UX, and relentless developer ingenuity. Additionally, new devices continue to flood the market, introducing new screen sizes/dimensions and operating systems that impact the compatibility and standardization of browser support.

Website design and interaction have become a tour de force, creating the need to break burgeoning html, css and javascript files into manageable, modular pieces. File sizes have also become cumbersome, and the need to optimize them for better page loading is now a necessity.

Use the following assets, guidelines and resources to optimize the front end experience by means of:

  • Maintainablity
  • Integration
  • DRY (don't repeat yourself) coding
  • Validated and properly structured HTML
  • Modular CSS and JS
  • Responsive layouts
  • Accessibility
  • Browser targets and compatibility
  • Media content
  • Object/module building
  • Task management
