Responsive Layouts

In the past, it was common practice to build a separate mobile site to compliment the main site. This practice was flawed as two sites then had to be maintained instead of just one, doubling maintenance/update time and causing inconsistencies between them. A solution to the problem arrived in 2011 based on responsiveness. Responsive Web Design, created by Ethan Marcotte, is the practice of creating a flexible web page layout that optimally responds to the screen size device. The layout shifts in a fluid fashion to compensate for a smaller or larger screen widths. There are a number of articles, tutorials, and opinions on responsive standards. "Responsive Web Design: What It Is and How To Use It" by Kayla Knight is a great resource for learning the basics of responsive web design and is a must read.

This project was built on responsive basic standards as well as these best practices:

  • Use CSS (for example, @media (query) { /* css rules go here */ }) to create the layout breakpoints. Responsiveness should always be handled using CSS and written in stylesheets. Do not use device platforms for layout breakpoints because new devices on the market makes the breakpoint logic outdated. Avoid using JS for breakpoints.
  • Use screen width to create the media query. Avoid device orientation (landscape/portrait) logic in the media query.
  • Avoid using more than one breakpoint. The more breakpoints means much more design, coding, and maintenance time. One breakpoint is sufficient for 99% of website objects. The other 1% can be targeted with object specific width media queries.
  • The optimal breakpoint is between 768px and 769px, which will handle target most devices with the best layout.
  • Use percentages and max-widths to create flexible grids, layouts, and objects.
  • Incorporate mobile first strategy for writing responsive code. This means to write the object's base code, and then extend the larger size layout with a media query (or separate object_md.scss file when using SCSS).
  • When using SCSS, create separate .scss files of your object styling for each breakpoint layout ( for example: _modal.scss and _modal_md.scss). The two files will be combined in the compiled modern.css and legacy.css files.
  • See the SCSS code in Common Objects and JS Modules for responsive styling examples. Download this project for a better look at how responsive coding is handled using SCSS.
