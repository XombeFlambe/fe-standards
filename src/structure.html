Directory Structure

Proper organization for files is an integral part of creating and maintaining a project's front end statics directory. Download this project's statics directory for a detailed look (dist and node_modules sub-directories not included. dist directory is created with a task runner and node_modules directory is created by node.js). Below are two examples of the proper structure for statics directories.

Project Structure with SCSS and Gulp

Show All Hide All
  • statics
    • dist
      • Is built by Gulp and includes a minified/compiled version of the src directory. The dist directory is for production and backend integration. Do not work in this directory, any changes will be over written when the src directory is compiled.
    • node_modules
      • Is built by npm install (node.js) and includes the gulp application modules. Do not edit in this directory, make all edits via npm in the command line. Do not save this directory in version control.
    • src
      • css
        • css (compiled from scss directory)
          • legacy.css
          • modern.css
        • scss
          • base
            • _elements.scss
            • _fonts.scss
            • _hide.scss
            • _reset.scss
          • helpers
            • _utils.scss
            • _vars.scss
          • landmarks
            • footer
              • _footer.scss
              • _footer_md.scss
            • header
              • _header.scss
              • _header_md.scss
          • modules (objects go here)
            • accordion
